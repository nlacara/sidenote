Introduction

A subsubtitle

The goal of this small project is to create simple css code for placing running side notesThis is a side note. It is positioned with a right float. in html pages. Inspired by my brother, I have recently considered beginning to blog again in a small scale context, but wanted to do so in a layout that I though most comfortably communicated information to the reader. I like side notes as a way of communicating additional information or side thoughts to the reader, and so I designed this simple css code for handling side notes.

Inspiration

I've used margin notes to great effect in lecture materialsSee the lecture notes posted here. and physical handouts for presentations. Having footnotes at the bottom of a page is very inconvenient and can interrupt the flow of the content for the reader (especially given that much of my audience would consume these materials on devices like tablets and laptops). Of course one way to solve this problem is to remove notes entirely, or else cut down their number significantly. There's some value in doing that, but I find that notes outside the main text can often be very valuable. Consequently, I wanted to implement a similar functionality for use on the web.

  • For more background on this project, the discussion here.I originally became interested in the use of margin notes and side notes when I ran across the Tufte-LaTeX class, which attempts to implement the style and page layout of Edward R. Tufte and Richard Feynman.
  • I wanted to implement functionality similar to the marginnote package available for LaTeX, wherein one can simply use the \marginnote{} command to insert marginal notes in the text.
  • Using this package, I implemented fairly robust handling of margin notes and footnotes in my LaTeX style file for linguistics.

Implementation

Most of the work here is done with css. While I experimented using spans with absolute position attributes, I ultimately landed on using floating spans. First, the text must be limited to take up only a fraction of the available screen width:

I recommend limiting the maximum width of the text, as shown here, since reading longer lines of text is harder. div {
  width: 75%;
  max-width: 45em;
}

The width of the side note should fit in the leftover space to the right.As far as I can tell, if you use a percentage, which I hvae, this is calculated as part of the width of the main text, not the total screen width. The key to making this work is to introduce a negative margin whose absolute value is greater than the width of the side note:

span.note {
  float: right;
  width: 28%;
  margin-right: -32%;
  position: relative;
  text-align: left;
}

This makes it possible to simply surround the intended side note with <span>…</span> tags, making the behavior somewhat akin to a LaTeX environment.

Other features

This css file, as can be seen above, has a special class for code blocks, as well as a class for block quotes (though this is distinct from the html <blockquote> tag.
